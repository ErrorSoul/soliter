# Респонсив в Defold — как это работает

## 1. Проекция (fixed_fit_projection)

Игра спроектирована под 960×540. Это **виртуальное игровое поле** — все координаты карт, слотов заданы в этом пространстве.

```
Игровое поле (мировые координаты):
┌─────────────────────────────────────────────────────┐
│ (0,540)                                   (960,540) │
│                                                     │
│   [free1] [free2] [free3]    [flower] [b1] [b2] [b3]│
│                                                     │
│   [stack1][stack2][stack3][stack4][stack5]...         │
│    ┌───┐  ┌───┐  ┌───┐                             │
│    │5_r│  │3_b│  │d_g│  ...                         │
│    │4_g│  │2_r│  │7_b│                              │
│    └───┘  └───┘  └───┘                              │
│                                                     │
│ (0,0)                                     (960,0)   │
└─────────────────────────────────────────────────────┘
```

Окно браузера может быть любого размера. `fixed_fit_projection` показывает всё поле 960×540 целиком, вписывая его в окно.

### Алгоритм масштабирования

```
Окно = 1920×1080                     Окно = 800×800
zoom = min(1920/960, 1080/540)       zoom = min(800/960, 800/540)
     = min(2.0, 2.0) = 2.0                = min(0.833, 1.481) = 0.833

Масштаб: 960×2 = 1920               Масштаб: 960×0.833 = 800
         540×2 = 1080                         540×0.833 = 450

Offset: (1920-1920)/2 = 0           Offset X: (800-800)/2 = 0
        (1080-1080)/2 = 0                  Y: (800-450)/2 = 175

→ Идеальное попадание,              → Letterbox: 175px сверху и снизу
  без letterbox                        (чёрные полосы)
```

### Визуализация

```
Окно 1920×1080 (16:9 = 16:9):       Окно 800×800 (1:1 ≠ 16:9):
┌────────────────────────┐           ┌──────────────────┐
│                        │           │▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓│ ← letterbox
│   Игра на весь экран   │           │┌────────────────┐│     175px
│   zoom = 2.0           │           ││  Игра          ││
│                        │           ││  zoom = 0.833  ││ 450px
│                        │           │└────────────────┘│
└────────────────────────┘           │▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓│ ← letterbox
                                     └──────────────────┘

Окно 1600×400 (ультраширокое):
┌──────────────────────────────────────┐
│▓▓▓│                            │▓▓▓▓│
│▓▓▓│  Игра, zoom = 0.741       │▓▓▓▓│
│▓▓▓│  711×400                   │▓▓▓▓│
│▓▓▓│                            │▓▓▓▓│
└──────────────────────────────────────┘
  ↑ letterbox слева                ↑ letterbox справа
    445px                           444px
```

**Ключевой момент:** карта с позицией `(200, 300)` ВСЕГДА находится на `(200, 300)` в мировых координатах. Проекция просто масштабирует и сдвигает весь мир целиком.

## 2. Input: screen_to_world

Когда игрок кликает, браузер даёт **экранные координаты** (пиксели окна). Нужно конвертировать обратно в мировые.

```
Пример: Окно 800×800, zoom = 0.833, offset_y = 175

Игрок кликнул по карте, которая в мире на (200, 300).
На экране эта карта отображается на:
  screen_x = 200 × 0.833 + 0    = 166.6 px
  screen_y = 300 × 0.833 + 175  = 424.9 px

Обратное преобразование (screen_to_world):
  world_x = (166.6 - 0)   / 0.833 = 200  ✓
  world_y = (424.9 - 175) / 0.833 = 300  ✓
```

**Нюанс текущего кода:** в `cursor.script` функция `screen_to_world()` содержит строку `w = w / (w / DISPLAY_WIDTH)` что всегда даёт `w = 960`. Это значит zoom всегда = 1.0 и offset = 0. Работает только потому что Defold при `fixed_fit_projection` сам нормализует `action.x/y` в виртуальные координаты. Возможно `screen_to_world()` вообще лишняя — `action.x/y` уже в мировых координатах.

## 3. GUI — отдельная система

GUI и game objects живут в разных системах координат:

```
Game objects:                          GUI:
- Мировые координаты (0-960, 0-540)   - Экранные координаты
- Масштабируются проекцией             - Масштабируются adjust mode
- Управляются render.script            - Управляются GUI-системой Defold
- go.set_position()                    - gui.set_position()
```

### adjust_mode

```
ADJUST_FIT:     масштабирует ноду сохраняя пропорции (как fixed_fit для GO)
ADJUST_ZOOM:    масштабирует с обрезкой (без letterbox, но края обрезаются)
ADJUST_STRETCH: растягивает (искажает пропорции)

Для кнопок: ADJUST_FIT
Для фона-затемнения victory: ADJUST_STRETCH (должен покрыть ВСЁ)
```

### x_anchor / y_anchor

```
XANCHOR_NONE + YANCHOR_NONE:     позиция от центра (по умолчанию)
XANCHOR_LEFT + YANCHOR_TOP:      привязка к левому верхнему углу
XANCHOR_RIGHT + YANCHOR_BOTTOM:  привязка к правому нижнему углу
```

### Проблема без anchor

```
Кнопка Restart на (900, 520) — БЕЗ anchor:

16:9 экран (совпадает с дизайном):
┌─────────────────────────────┐
│                  [Restart]  │  ← Видна, всё ок
│                             │
│   Игровое поле              │
└─────────────────────────────┘

4:3 экран с letterbox:
┌────────────────────────────────────┐
│▓▓│                        [R] │▓▓▓│  ← Кнопка может оказаться
│▓▓│                            │▓▓▓│     ЗА letterbox! Не кликабельна.
│▓▓│   Игровое поле             │▓▓▓│
└────────────────────────────────────┘

С anchor XANCHOR_RIGHT:
┌────────────────────────────────────┐
│▓▓│                     [Restart]▓▓│  ← Привязана к правому краю
│▓▓│                              ▓▓│     ВИДИМОГО пространства
│▓▓│   Игровое поле               ▓▓│
└────────────────────────────────────┘
```

## 4. Проблемы и решения

### Проблема 1: Клик в letterbox

Игрок кликает в чёрную полосу. `screen_to_world` вернёт координаты за пределами 0-960 / 0-540. Карты там нет → ничего не происходит. Не баг, но нужно убедиться что `screen_to_world` корректно работает при letterbox.

### Проблема 2: Victory overlay не покрывает letterbox

```
┌──────────────────────────────┐
│▓▓│████████████████████│▓▓▓▓▓│  ████ = victory overlay (ADJUST_FIT)
│▓▓│████ YOU WIN! ██████│▓▓▓▓▓│  ▓▓▓▓ = letterbox (НЕ затемнён!)
│▓▓│████████████████████│▓▓▓▓▓│
└──────────────────────────────┘
```

**Решение:** для фона-затемнения использовать `ADJUST_STRETCH`.

### Проблема 3: Цвет letterbox

Сейчас letterbox = чёрный. Нужно заменить на цвет стола в `render.script`:
```lua
render.clear({[render.BUFFER_COLOR_BIT] = vmath.vector4(0.1, 0.2, 0.1, 1), ...})
```

### Проблема 4: HTML5 canvas не заполняет iframe

Яндекс Игры загружают игру в iframe. По умолчанию canvas = 960×540 пикселей.

```
Браузер 1920×1080:
┌──────────────────────────────────────┐
│                                      │
│         ┌──────────┐                 │  ← canvas 960×540
│         │  Игра    │                 │     маленькое окно!
│         └──────────┘                 │
│                                      │
└──────────────────────────────────────┘

Нужно:
┌──────────────────────────────────────┐
│ ┌──────────────────────────────────┐ │  ← canvas заполняет
│ │           Игра                   │ │     весь iframe
│ └──────────────────────────────────┘ │
└──────────────────────────────────────┘
```

**Решение:** кастомный HTML-шаблон + CSS `width: 100vw; height: 100vh`.

### Проблема 5: Retina / HiDPI

На Retina `window.get_size()` возвращает физические пиксели (1920×1080), а `action.x/y` — логические. Нужно тестировать и при необходимости учитывать `window.get_display_scale()`.

## 5. Итоговая схема

```
                         ┌─────────────────┐
                         │   Окно браузера  │
                         │   (любой размер) │
                         └────────┬────────┘
                                  │
                    ┌─────────────┴─────────────┐
                    │                           │
              ┌─────┴──────┐              ┌─────┴──────┐
              │  Render    │              │    GUI     │
              │  Pipeline  │              │   System   │
              └─────┬──────┘              └─────┬──────┘
                    │                           │
           fixed_fit_projection          adjust_mode + anchors
           zoom + offset + letterbox     привязка к краям экрана
                    │                           │
              ┌─────┴──────┐              ┌─────┴──────┐
              │ Game Objects│              │  GUI Nodes │
              │ (карты,     │              │ (кнопки,   │
              │  слоты)     │              │  оверлеи)  │
              │ 960×540     │              │            │
              └─────────────┘              └────────────┘
                    │
              ┌─────┴──────┐
              │   Input    │
              │ action.x,y │──→ screen_to_world() ──→ hit test
              └────────────┘     (возможно лишняя)
```

**Главное правило:** game objects не знают про размер окна. Вся адаптация — в проекции и GUI-системе. Код игры работает только с координатами 0-960 / 0-540.
